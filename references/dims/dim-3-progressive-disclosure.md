# Dim 3 — Progressive Disclosure

Skills are folders, not files. The file system is context engineering.

**Score 5 — Excellent**
- SKILL.md is the hub — points to other files for depth
- Has subdirectories: `references/`, `scripts/`, `assets/`
- Heavy content (API docs, templates, examples) is in sub-files
- SKILL.md itself is <500 lines

**Score 4 — Good**
- Has at least one sub-directory with referenced files
- SKILL.md is the hub but a few sections could still be extracted

**Score 3 — Acceptable**
- Has at least one sub-file referenced
- SKILL.md is somewhat long but manageable

**Score 2 — Weak**
- Has a sub-file but SKILL.md is still bloated with inline content that should be extracted

**Score 1 — Poor**
- Everything is in a single flat SKILL.md
- No sub-files, even though the skill has significant depth
- SKILL.md is bloated (>500 lines) with embedded reference content

**Key questions:**
- [ ] Is the skill a folder with multiple files?
- [ ] Does SKILL.md reference sub-files clearly?
- [ ] Is heavy content (examples, API docs) moved out of the main file?
